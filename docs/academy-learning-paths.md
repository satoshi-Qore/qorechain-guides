# Academy Learning Paths / Academy Öğrenme Yolları

This page organizes QoreChain Guides by user role. It helps new readers choose the right starting point without reading the entire repository first.

Bu sayfa, QoreChain Guides içeriğini kullanıcı rolüne göre düzenler. Yeni gelenlerin tüm depoyu okumadan doğru başlangıç noktasını seçmesine yardımcı olur.

## Purpose / Amaç

QoreChain Guides is growing into a bilingual community knowledge hub. As the repository expands, users need clear learning paths for different goals: learning the basics, running a Light Node, completing Eigenstate 2 tasks, contributing documentation, or preparing for future developer resources.

QoreChain Guides iki dilli bir topluluk bilgi merkezine dönüşmektedir. Depo büyüdükçe kullanıcıların farklı hedefler için net öğrenme yollarına ihtiyacı olur: temel bilgileri öğrenmek, Light Node çalıştırmak, Eigenstate 2 görevlerini tamamlamak, dokümantasyona katkı vermek veya gelecekteki geliştirici kaynaklarına hazırlanmak.

## Path 1: New User / Yeni Kullanıcı

### Goal / Amaç

Build a basic understanding of QoreChain terms, community resources, and common questions.

QoreChain terimlerini, topluluk kaynaklarını ve sık sorulan soruları temel seviyede anlamak.

### Suggested Order / Önerilen Sıra

1. [EN Knowledge Base](../en/knowledge-base.md) / [TR Bilgi Bankası](../tr/bilgi-bankasi.md)
2. [EN FAQ](../en/faq.md) / [TR SSS](../tr/sss.md)
3. [EN Glossary](../en/glossary.md) / [TR Terimler Sözlüğü](../tr/terimler-sozlugu.md)
4. [Roadmap](../ROADMAP.md)

### Outcome / Çıktı

The reader should understand the repository structure, common terminology, and where to find role-specific information.

Okuyucu depo yapısını, temel terminolojiyi ve role göre hangi bilgiye nereden ulaşacağını anlamış olmalıdır.

## Path 2: Light Node Operator / Light Node Operatörü

### Goal / Amaç

Follow practical setup, verification, monitoring, and troubleshooting notes for Light Node-related workflows.

Light Node ile ilgili kurulum, doğrulama, takip ve sorun giderme notlarını pratik şekilde takip etmek.

### Suggested Order / Önerilen Sıra

1. [EN Light Node Guide](../en/light-node/) / [TR Light Node Rehberi](../tr/light-node/)
2. [EN Troubleshooting](../en/troubleshooting.md) / [TR Hata Çözümleri](../tr/sorun-giderme.md)
3. [EN FAQ](../en/faq.md) / [TR SSS](../tr/sss.md)
4. [Support](../SUPPORT.md)

### Outcome / Çıktı

The reader should be able to follow a documented operator workflow and know where to look when a problem appears.

Okuyucu belgelenmiş bir operatör akışını takip edebilmeli ve sorun çıktığında hangi sayfaya bakacağını bilmelidir.

## Path 3: Eigenstate 2 Participant / Eigenstate 2 Katılımcısı

### Goal / Amaç

Understand the observed Eigenstate 2 task categories, proof submission expectations, and safe proof-link practices.

Gözlemlenen Eigenstate 2 görev kategorilerini, kanıt gönderimi beklentilerini ve güvenli kanıt linki pratiklerini anlamak.

### Suggested Order / Önerilen Sıra

1. [EN Eigenstate 2 Tasks](../en/eigenstate-2-tasks.md) / [TR Eigenstate 2 Görevleri](../tr/eigenstate-2-gorevleri.md)
2. [EN Proof Submission Guide](../en/eigenstate-2-proof-submission.md) / [TR Kanıt Gönderimi](../tr/eigenstate-2-kanit-gonderimi.md)
3. [EN Proof Link Guide](../en/proof-link-guide.md) / [TR Kanıt Linki Rehberi](../tr/kanit-linki-rehberi.md)
4. Category guide based on the task type / Görev türüne göre kategori rehberi

### Outcome / Çıktı

The reader should understand how to interpret task requirements, avoid exposing private information, and prepare review-friendly proof links.

Okuyucu görev gereksinimlerini yorumlamayı, özel bilgileri paylaşmamayı ve incelemeye uygun kanıt linkleri hazırlamayı anlamış olmalıdır.

## Path 4: Community Contributor / Topluluk Katkıcısı

### Goal / Amaç

Help maintain or improve the repository without breaking bilingual structure, file naming, or documentation style.

İki dilli yapıyı, dosya adlandırmasını veya dokümantasyon stilini bozmadan depoyu korumak veya iyileştirmek.

### Suggested Order / Önerilen Sıra

1. [Contributing](../CONTRIBUTING.md)
2. [Style Guide](../STYLE_GUIDE.md)
3. [Documentation Standards](./documentation-standards.md)
4. [First Contribution](../contributing/first-contribution.md)
5. [Pull Request Template Notes](../contributing/pull-request-template.md)

### Outcome / Çıktı

The contributor should know how to propose changes, keep Turkish and English pages aligned, and avoid unsupported claims.

Katkıcı değişiklik önermeyi, Türkçe ve İngilizce sayfaları uyumlu tutmayı ve desteklenmeyen iddialardan kaçınmayı bilmelidir.

## Path 5: Technical Reader / Teknik Okuyucu

### Goal / Amaç

Review operational documentation quality, identify gaps, and prepare future developer or infrastructure-oriented contributions.

Operasyon dokümantasyonu kalitesini incelemek, eksikleri belirlemek ve gelecekteki geliştirici veya altyapı odaklı katkılara hazırlanmak.

### Suggested Order / Önerilen Sıra

1. [Documentation Standards](./documentation-standards.md)
2. [Roadmap](../ROADMAP.md)
3. [EN Troubleshooting](../en/troubleshooting.md) / [TR Hata Çözümleri](../tr/sorun-giderme.md)
4. [QoreChain Tools](https://github.com/satoshi-Qore/qorechain-tools)

### Outcome / Çıktı

The reader should understand the repository's documentation model and where future technical resources can connect.

Okuyucu deponun dokümantasyon modelini ve gelecekte teknik kaynakların nereye bağlanabileceğini anlamış olmalıdır.

## Maintenance Notes / Bakım Notları

- Keep this page aligned with `README.md`, `en/README.md`, `tr/README.md`, and `ROADMAP.md`.
- Add new paths only when there is enough content to support them.
- Avoid promising final mainnet behavior, guaranteed rewards, or fixed network parameters.
- Link critical or changing information back to official QoreChain sources when available.

- Bu sayfayı `README.md`, `en/README.md`, `tr/README.md` ve `ROADMAP.md` ile uyumlu tutun.
- Yeni yolları yalnızca onları destekleyecek yeterli içerik olduğunda ekleyin.
- Nihai mainnet davranışı, garanti ödül veya sabit ağ parametresi vaat etmeyin.
- Kritik veya değişebilecek bilgileri mümkün olduğunda resmi QoreChain kaynaklarına yönlendirin.
