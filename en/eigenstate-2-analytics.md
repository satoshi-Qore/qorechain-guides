# Eigenstate 2 Analytics Tasks

This guide covers the Analytics category in QoreChain Eigenstate 2.

Observed category size: 21 tasks.

Analytics tasks usually involve market-data, exchange, watchlist, community insight, and discussion platforms such as CoinMarketCap, CoinGecko, MEXC Insight, Binance Square, and similar services.

## Purpose

The goal of Analytics tasks is to increase discoverability and discussion quality around QoreChain on data and exchange-related platforms.

## Common Task Types

| Task Type | Typical Proof | Notes |
|---|---|---|
| Add to watchlist | Profile/page link or screenshot proof | Watchlist actions may not create public links |
| Follow a project page | Project page link and short description | Use the exact platform requested by the task |
| Like posts | Post link, proof page, or screenshot link | Likes may not generate unique proof links |
| Comment on posts | Direct post/comment link | Keep comments relevant and at least the requested length |
| Create a 50+ or 100+ word post | Public post link | Avoid price predictions or unsupported claims |
| Vote or participate in polls | Poll link, proof page, or screenshot link | Use public proof when the vote is not directly visible |

## Recommended Workflow

1. Open the exact task from the Analytics category.
2. Identify the target platform.
3. Complete the requested action on the correct account.
4. Copy the post, profile, or project page link.
5. If no direct proof exists, use a screenshot link or GitHub proof note.
6. Submit a short description.

## Content Guidance

For longer posts, focus on:

- post-quantum security as a research direction;
- Light Node infrastructure as a community learning topic;
- documentation and onboarding;
- responsible pre-mainnet language;
- ecosystem education rather than promotion.

## Description Examples

```text
I added QoreChain to my watchlist and submitted a public proof link for manual review.
```

```text
I wrote a 100+ word post about QoreChain on the requested analytics platform and submitted the post link.
```

```text
I commented on the requested QoreChain post with a relevant response and submitted the post link for review.
```

## Risk Notes

- Do not submit private account pages that reviewers cannot open.
- Do not promise rewards, price movement, or guaranteed outcomes.
- Do not use fake engagement or repeated generic comments.
