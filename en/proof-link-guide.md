# Proof Link Guide

This guide explains how to submit evidence when the QoreChain task panel does not provide a screenshot upload field.

Some tasks only provide two fields:

- Proof link
- Describe what you did

In this case, a screenshot cannot be uploaded directly. The evidence should be converted into a public or shareable link that a reviewer can open.

## Basic Flow

1. Complete the task.
2. Take a screenshot if needed.
3. Upload the evidence somewhere public or accessible by link.
4. Paste the link into the `proof link` field.
5. Write a short and clear note in the `Describe what you did` field.

## Proof Link Options

| Option | When It Is Useful | Note |
|---|---|---|
| GitHub proof repository | Tracking many tasks in an organized way | One of the cleanest and strongest options for technical users |
| GitHub Gist | A single task or short proof note | Simple and lightweight |
| Google Drive | Sharing screenshots | Access should be set to `anyone with the link can view` |
| X or LinkedIn post | Social tasks | Works well when the proof is naturally social |
| Imgur or similar image host | Creating a screenshot link only | The link should be publicly viewable |
| Public Telegram channel message | Community sharing tasks | The channel or message must be public |

## GitHub Proof Repository Method

This method is useful for users who want to keep task evidence organized over time.

Suggested structure:

```text
qorechain-task-proofs/
|-- README.md
|-- eigenstate-2/
|   |-- linkedin-follow.md
|   |-- github-follow.md
|   |-- reddit-intro.md
|-- screenshots/
    |-- linkedin-follow.png
    |-- github-follow.png
```

A short Markdown proof page can be created for each task:

```md
# Follow QoreChain Association on LinkedIn

Task: Follow QoreChain Association on LinkedIn
Status: Completed
Evidence: Screenshot attached in this repository
Date: YYYY-MM-DD

I followed the QoreChain Association LinkedIn page. Since the task form does not include a screenshot upload field, I am submitting this public GitHub proof link for manual review.
```

## Google Drive Method

1. Upload the screenshot to Google Drive.
2. Open the file sharing settings.
3. Set access to `anyone with the link can view`.
4. Copy the link and paste it into the task panel's `proof link` field.

Make sure the link is not private or restricted.

## Description Examples

For general tasks:

```text
I completed the requested task. Since there is no screenshot upload field, I provided a public proof link where the evidence can be reviewed manually.
```

For follow tasks:

```text
I followed the requested QoreChain account/page and provided a public proof link for manual review.
```

For like or engagement tasks:

```text
I completed the required engagement action and provided a public proof link showing the completed task.
```

For GitHub tasks:

```text
I completed the requested GitHub action and provided a public GitHub link as proof for manual review.
```

## Safety Notes

- The proof link must be accessible to the reviewer.
- Private accounts, closed groups, or login-only pages may be weak evidence.
- Do not share seed phrases, private keys, wallet secrets, email addresses, or phone numbers.
- Hide sensitive information in screenshots when needed.
- Avoid exaggerated claims or approval guarantees in the description.

## Short Summary

If there is no screenshot upload field, convert the evidence into a link. The reviewer should be able to open the link and understand that the task was completed.
