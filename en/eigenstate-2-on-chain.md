# Eigenstate 2 On-Chain Tasks

This guide covers the On-Chain category in QoreChain Eigenstate 2.

Observed category size: 23 tasks.

On-Chain tasks usually involve wallet actions, QOR holdings, staking, delegation, governance, Light Node registration, transactions, bridge actions, and other network-related activity.

## Purpose

The goal of On-Chain tasks is to verify real network participation. These tasks can be more sensitive than social or educational tasks because they may involve wallets, balances, nodes, or transactions.

## Common Task Types

| Task Type | Typical Proof | Notes |
|---|---|---|
| Send transaction | Transaction hash or explorer link | Verify the correct network before submitting |
| Hold QOR | Wallet/task panel proof | Avoid exposing unnecessary wallet details |
| Delegate or stake | Transaction/explorer proof | Check official requirements first |
| Vote on governance | Proposal or vote proof | Confirm the proposal is real and current |
| Register Light Node | Node panel, official task proof, or screenshot link | Hide server secrets and private data |
| Run Light Node | Panel or monitoring proof | Do not expose private IPs or credentials if sensitive |
| Bridge action | Bridge transaction proof | Confirm chain, asset, and bridge route carefully |

## Safety Checklist

Before doing any on-chain task:

- confirm the official task instruction;
- confirm the network and wallet;
- never share seed phrases or private keys;
- do not reveal unnecessary wallet or server data;
- verify transaction details before signing;
- keep proof links public but privacy-aware.

## Description Examples

```text
I completed the requested on-chain action and submitted the related transaction or proof link for manual review.
```

```text
I registered and ran a QoreChain Light Node, then submitted a public proof link for review.
```

```text
I delegated QOR according to the task requirement and submitted the relevant proof link.
```

## Mainnet Caution

If mainnet is not active or a task depends on a future network state, do not claim that a mainnet-only action is complete. Use careful wording and wait for official instructions.

## Risk Notes

- Do not share private keys, seed phrases, full server credentials, API tokens, or login data.
- Do not trust unofficial RPC, bridge, or staking links without verification.
- Do not submit fabricated transaction data.
