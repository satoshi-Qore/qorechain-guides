# QoreChain Glossary

This glossary explains common terms used in QoreChain discussions, community guides, and research-oriented documentation. It is written for users who want a practical introduction to blockchain, infrastructure, and post-quantum terminology.

This page is educational and community-maintained. It does not replace official QoreChain documentation.

## Core Terms

### QoreChain

QoreChain is described as a Layer 1 blockchain infrastructure project focused on post-quantum security, AI-native infrastructure, and multi-VM execution. In this glossary, QoreChain is treated as a research and ecosystem subject rather than as a product claim.

### Layer 1

A Layer 1 blockchain is a base network that processes transactions, maintains its own consensus mechanism, and provides the foundation for applications, tokens, smart contracts, and network participation.

### Post-Quantum Security

Post-quantum security refers to cryptographic approaches intended to remain secure against both classical computers and future quantum computers. In blockchain systems, this topic is important because wallets, signatures, and assets may need long-term protection.

### Quantum-Resistant Cryptography

Quantum-resistant cryptography is a category of cryptographic methods designed to resist attacks from quantum algorithms. Examples in broader research include lattice-based, hash-based, code-based, and multivariate approaches.

### AI-Native Infrastructure

AI-native infrastructure refers to systems where artificial intelligence is considered part of the design, tooling, monitoring, or operational workflow. In blockchain contexts, AI may assist with smart contract review, network monitoring, documentation search, and troubleshooting.

### Multi-VM Architecture

Multi-VM architecture means supporting more than one virtual machine or execution environment. This may allow developers from different ecosystems to build applications using familiar tools, but it also creates additional security and interoperability questions.

### Virtual Machine

A virtual machine is an execution environment where smart contracts or programs run. In blockchain systems, virtual machines define how code is executed, how fees are calculated, and how state changes are validated.

### EVM

EVM stands for Ethereum Virtual Machine. It is one of the most widely used smart contract execution environments and is supported by many blockchain ecosystems.

### CosmWasm

CosmWasm is a smart contract platform used in the Cosmos ecosystem. It allows developers to write contracts, often using WebAssembly-based execution.

### SVM

SVM generally refers to the Solana Virtual Machine or Solana-style execution environment. It is associated with high-throughput application execution and a different design model from the EVM.

## Network and Infrastructure Terms

### Node

A node is a machine that participates in a blockchain network by storing data, communicating with peers, validating information, or supporting access to network services.

### Light Node

A light node is usually a lighter form of network participation compared with a full validator or full node. Its exact role depends on the network design. For QoreChain, light node details should be checked against current official documentation and task requirements.

### Validator

A validator is a network participant responsible for validating transactions, proposing or confirming blocks, and supporting consensus. Validator requirements and incentives vary by blockchain.

### Delegation

Delegation allows token holders to support a validator without running validator infrastructure themselves. Delegation mechanics depend on the network's staking and governance model.

### RPC Endpoint

An RPC endpoint is a network access point that applications, wallets, or tools use to communicate with a blockchain node. RPC quality affects reliability, synchronization, and user experience.

### Explorer

A blockchain explorer is a tool that lets users inspect transactions, blocks, addresses, validators, and other network data.

### Faucet

A faucet is a tool that distributes small amounts of test tokens or network tokens, usually for testing, onboarding, or development purposes.

### Uptime

Uptime measures how consistently a node, validator, or service remains online. High uptime is important for reliable infrastructure participation.

## Token and Governance Terms

### QOR

QOR is the token associated with the QoreChain ecosystem. Its confirmed uses, distribution, and long-term economic role should be evaluated using official documentation and observable network data.

### Tokenomics

Tokenomics refers to the economic design of a blockchain token, including supply, distribution, utility, incentives, staking, governance, and sustainability.

### Staking

Staking is the process of locking or committing tokens to support network security, validator participation, or governance. The exact rules depend on the network.

### Governance

Governance refers to how network decisions are proposed, discussed, voted on, and implemented. Governance may include on-chain voting, community discussion, validator coordination, and protocol updates.

### Proposal

A proposal is a suggested network change or governance action. Depending on the blockchain, proposals may require token deposits, validator support, or community voting.

## Community and Documentation Terms

### Community Guide

A community guide is documentation created by ecosystem participants to help users understand setup steps, terminology, troubleshooting, or task workflows. It should avoid unsupported claims and point users to official sources for critical decisions.

### Mainnet

Mainnet is the live production network where real network activity takes place. Before mainnet, many claims should be treated as preliminary and should be updated after official launch and observable data.

### Testnet

A testnet is a testing network used for experimentation, development, onboarding, and infrastructure preparation before or alongside mainnet.

### Quest or Task

A quest or task is an ecosystem activity that users complete for points, review, or participation tracking. Task requirements should be followed exactly, and proof should be submitted according to the platform instructions.

## Research Note

Many QoreChain-related terms may evolve as documentation, mainnet deployment, governance processes, and ecosystem tooling mature. This glossary should be updated when official information changes or new technical details become available.
