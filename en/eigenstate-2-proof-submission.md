# Eigenstate 2 Proof Submission Guide

This guide explains how to prepare and submit proof for QoreChain Eigenstate 2 tasks.

Most task forms include:

- `proof link`
- `Describe what you did`

Some tasks do not provide a screenshot upload field. In those cases, the evidence must be converted into a public or shareable link.

## Proof Decision Tree

| Task Situation | Recommended Proof |
|---|---|
| Public post or comment exists | Direct post/comment link |
| GitHub issue or PR exists | Issue or PR link |
| Transaction exists | Explorer or transaction link |
| Follow/like/watch action has no public proof | Screenshot uploaded to a public proof page |
| Screenshot is required but no upload field exists | GitHub proof note, Gist, Drive link, or image host link |
| Task was completed inside the task panel | Short description plus relevant official page link |

## Recommended Proof Page Format

```md
# Task Title

Task: [exact task name]
Category: [Social / Analytics / Development / Education / Regional / On-Chain]
Status: Completed
Date: YYYY-MM-DD
Proof: [screenshot, link, transaction, post, or note]

I completed the requested task and provided this public proof page because the task form does not include a screenshot upload field.
```

## Description Field Template

```text
I completed the requested QoreChain Eigenstate 2 task. Since this task is reviewed manually, I submitted a public proof link and a short description for verification.
```

## Strong Proof Checklist

Before submitting, check:

- The proof link opens in a private browser window.
- The link points to the correct task, post, profile, transaction, or proof page.
- The description matches the task instruction.
- Screenshots do not expose private keys, seed phrases, email addresses, phone numbers, API keys, or server credentials.
- The wording does not claim approval before review is complete.

## Weak Proof Examples

Avoid submitting:

- private social profile links that reviewers cannot open;
- screenshots with hidden task context;
- unrelated homepage links;
- broken Drive links;
- copied descriptions that do not match the task;
- claims such as `approved`, `guaranteed`, or `official` without evidence.

## Best Public Proof Options

| Option | Strength | Best Use |
|---|---|---|
| GitHub proof repository | High | Many tasks, organized proof archive |
| GitHub Gist | Medium | Single proof note |
| Google Drive public link | Medium | Screenshots |
| X or LinkedIn post | Medium | Social tasks |
| Image host | Medium | Screenshot-only evidence |
| Public Telegram message | Low to medium | Community sharing tasks |
