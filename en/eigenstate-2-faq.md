# Eigenstate 2 FAQ

This FAQ answers common questions about QoreChain Eigenstate 2 community tasks.

## How many Eigenstate 2 tasks are there?

The observed task set contains 149 tasks across six categories:

| Category | Tasks |
|---|---:|
| Social | 79 |
| Analytics | 21 |
| Development | 6 |
| Education | 8 |
| Regional | 12 |
| On-Chain | 23 |

Task counts may change if the official task panel is updated.

## Is this an official guide?

No. This is a community-maintained guide. Always follow the official QoreChain task panel for final instructions.

## What should I submit if there is no screenshot upload field?

Convert the screenshot or evidence into a public proof link. You can use a GitHub proof repository, GitHub Gist, Google Drive public link, image host, or public social post.

## What should I write in `Describe what you did`?

Write one short sentence explaining the action you completed.

Example:

```text
I completed the requested QoreChain task and submitted a public proof link for manual review.
```

## Can I submit a screenshot link?

Yes, if the reviewer can open the link and the screenshot clearly shows the completed task. Hide sensitive information first.

## Can I submit a profile link for follow tasks?

Sometimes, but a profile link alone may be weak if the follow action is not visible. A public proof page with a screenshot may be stronger.

## Can I claim a PR is merged before it is merged?

No. Use `open pull request` or `submitted pull request` until GitHub shows it as merged.

## What if a task depends on mainnet?

Do not claim completion before the required network state is available. Wait for official instructions and use careful wording.

## What information should I never share?

Never share seed phrases, private keys, API keys, login tokens, full server credentials, or unnecessary personal data.

## Which category should I start with?

Start with lower-risk tasks first: Education, Social, Analytics, and Development tasks. On-Chain tasks should be handled more carefully because they may involve wallets, transactions, nodes, or staking.
