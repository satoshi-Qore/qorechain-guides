# QoreChain Guides Katki Rehberi / Contributing Guide

QoreChain Guides'i gelistirmeye yardim ettiginiz icin tesekkurler. Bu depo; teknik kaynaklar, node operasyonlari, gorev surecleri ve sorun giderme notlari icin topluluk tarafindan hazirlanan bir dokumantasyon merkezidir.

Thank you for helping improve QoreChain Guides. This repository is a community-maintained documentation hub for technical resources, node operations, task workflows, and troubleshooting notes.

## Neler Katki Olarak Eklenebilir?

- Kurulum iyilestirmeleri
- Light Node operasyon notlari
- Sorun giderme adimlari
- Sik sorulan sorular
- Turkce veya Ingilizce ceviriler
- Belirsiz ya da guncelligini yitirmis dokumantasyon duzeltmeleri
- Yeni gorev akisi notlari

## What You Can Contribute

- Setup improvements
- Light Node operation notes
- Troubleshooting steps
- Frequently asked questions
- Turkish or English translations
- Corrections for unclear or outdated documentation
- New task workflow notes

## Dokumantasyon Stili

- Adimlari kisa, uygulanabilir ve kontrol edilebilir tutun.
- Kontroller, durumlar ve sorun giderme akislarinda tablo kullanin.
- Komutlar icin kod bloklari kullanin.
- Tahminden kacinin. Bir kural degisebilecekse kullanicilari resmi kaynaklara yonlendirin.
- Mumkun oldugunda Turkce ve Ingilizce sayfalari ayni yapida tutun.
- Topluluk okuyuculari icin sade bir dil kullanin.

## Documentation Style

- Keep steps short, practical, and easy to verify.
- Use tables for checks, statuses, and troubleshooting flows.
- Use fenced code blocks for commands.
- Avoid speculation. If a rule may change, point users to official sources.
- Keep Turkish and English pages aligned when possible.
- Use plain language for community readers.

## Iki Dilli Yapi

Turkce sayfalar `tr/` klasoru altinda yer alir.

Ingilizce sayfalar `en/` klasoru altinda yer alir.

Yeni bir konu eklerken mumkunse iki dildeki karsiligini da ekleyin veya planlayin. Sadece bir dil hazirsa, diger dildeki sayfanin daha sonra eklenecegini belirtin.

## Bilingual Structure

Turkish pages live under `tr/`.

English pages live under `en/`.

When adding a new topic, try to add or plan both language versions. If only one language is ready, mention that the matching page is still planned.

## Onerilen Akis

1. Buyuk degisiklikler veya yeni konular icin once issue acin.
2. Pull request'leri tek bir konuya odakli tutun.
3. Ne degistigini ve neden degistigini aciklayin.
4. Gondermeden once linkleri kontrol edin.
5. Komutlarin dogru baglamda verildiginden emin olun.

## Suggested Workflow

1. Open an issue for large changes or new topics.
2. Keep pull requests focused on one topic.
3. Explain what changed and why.
4. Check links before submitting.
5. Make sure commands are placed in the correct context.

## Guvenlik Notu

Bu depo resmi QoreChain dokumantasyonunun yerine gecmez. Kritik kurulum, stake, ag veya guvenlik adimlari icin kullanicilar resmi kaynaklari ve guncel duyurulari kontrol etmelidir.

## Safety Note

This repository does not replace official QoreChain documentation. For critical setup, stake, network, or security steps, users should check official sources and current announcements.
