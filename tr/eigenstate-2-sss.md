## Proof link alanına ne yazmalıyım?

Görev doğrudan link üretiyorsa o linki yazın. Örneğin GitHub issue, PR, Reddit post, Medium yazısı veya explorer transaction linki.

Görev link üretmiyorsa ilgili resmi sayfa linkini ve kısa açıklamayı kullanın. Ekran görüntüsü gerekiyorsa ekran görüntüsünü erişilebilir bir sayfaya koyup o sayfanın linkini ekleyebilirsiniz.

## Ekran görüntüsü yükleme alanı yoksa ne yapmalıyım?

Ekran görüntüsünü doğrudan yükleyemiyorsanız, onu güvenli bir kanıt linkine dönüştürmeniz gerekir. Örneğin GitHub proof reposu, GitHub gist, public belge veya sosyal medya paylaşımı kullanılabilir.

Hassas bilgileri mutlaka kapatın.

## Beğeni veya takip görevlerinde kanıt nasıl olur?

Beğeni ve takip görevleri çoğu zaman public link üretmez. Bu durumda:

- ilgili resmi sayfa veya gönderi linkini ekleyin;
- açıklamada ne yaptığınızı yazın;
- gerekiyorsa ekran görüntüsü kanıtını linke dönüştürün.

## Describe what you did ne demek?

Bu alan, görevi nasıl tamamladığınızı kısa şekilde anlatmanız içindir.

Örnek:

```text
I followed the official QoreChain page on LinkedIn and submitted the page link for manual review.
```

## GitHub PR açıkken merged diyebilir miyim?

Hayır. PR yalnızca GitHub üzerinde merge edildiyse "merged PR" denmelidir.

Açık PR için daha doğru ifade:

```text
I submitted an open pull request for manual review.
```

## Meaningful issue ne demek?

Meaningful issue, gerçek bir eksikliği, belirsizliği veya iyileştirme önerisini açıklayan issue demektir. Sadece "good project" veya "please update" gibi genel ifadeler yeterli olmayabilir.

## Mainnet başlamadıysa on-chain görevleri yapmalı mıyım?

Mainnet, TGE, staking, bridge veya validator gibi görevlerde resmi duyuru ve görev paneli netleşmeden acele edilmemelidir. Yanlış ağda işlem yapmak veya eksik bilgiyle hareket etmek risklidir.

## Aynı metni her platformda kullanabilir miyim?

Kullanmak mümkün olsa da önerilmez. Aynı metin spam gibi görünebilir. Her platform için kısa, doğal ve bağlama uygun bir metin kullanmak daha iyidir.

## Görev onayını garanti edebilir miyiz?

Hayır. Görevler manuel incelenir ve onay kararı ekip tarafından verilir. Bu rehber yalnızca kanıtı daha düzenli hazırlamaya yardımcı olur.
