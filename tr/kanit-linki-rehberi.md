# Kanit Linki Rehberi

Bu rehber, QoreChain gorev panelinde ekran goruntusu yukleme alani olmadiginda kanitin nasil linke donusturulebilecegini aciklar.

Bazi gorevlerde yalnizca iki alan bulunur:

- Proof link
- Describe what you did

Bu durumda ekran goruntusu dogrudan yuklenemez. Kanit, inceleyicinin erisebilecegi herkese acik veya paylasilabilir bir link haline getirilmelidir.

## Temel Akis

1. Gorevi tamamlayin.
2. Gerekirse ekran goruntusu alin.
3. Kaniti herkese acik veya link ile gorulebilir bir yere yukleyin.
4. Olusan linki `proof link` alanina ekleyin.
5. `Describe what you did` alanina kisa ve net bir aciklama yazin.

## Kullanilabilecek Kanit Linki Secenekleri

| Secenek | Ne Zaman Uygun? | Not |
|---|---|---|
| GitHub proof repository | Cok sayida gorevi duzenli takip etmek icin | En duzenli ve teknik olarak guclu seceneklerden biridir |
| GitHub Gist | Tek bir gorev veya kisa kanit notu icin | Basit ve hafif bir alternatiftir |
| Google Drive | Ekran goruntusu paylasmak icin | Paylasim ayari `linke sahip olan goruntuleyebilir` olmalidir |
| X veya LinkedIn paylasimi | Sosyal gorevlerde | Kanit sosyal baglamla uyumlu olur |
| Imgur veya benzeri gorsel yukleme servisi | Sadece ekran goruntusu linki uretmek icin | Link herkese acik gorunebilmelidir |
| Herkese acik Telegram kanal mesaji | Topluluk paylasimlari icin | Kanal veya mesaj herkese acik olmalidir |

## GitHub Proof Repo Yontemi

Bu yontem, gorev kanitlarini uzun vadeli ve duzenli tutmak isteyen kullanicilar icin uygundur.

Onerilen yapi:

```text
qorechain-task-proofs/
|-- README.md
|-- eigenstate-2/
|   |-- linkedin-follow.md
|   |-- github-follow.md
|   |-- reddit-intro.md
|-- screenshots/
    |-- linkedin-follow.png
    |-- github-follow.png
```

Her gorev icin kisa bir Markdown sayfasi hazirlanabilir:

```md
# Follow QoreChain Association on LinkedIn

Task: Follow QoreChain Association on LinkedIn
Status: Completed
Evidence: Screenshot attached in this repository
Date: YYYY-MM-DD

I followed the QoreChain Association LinkedIn page. Since the task form does not include a screenshot upload field, I am submitting this public GitHub proof link for manual review.
```

## Google Drive Yontemi

1. Ekran goruntusunu Google Drive'a yukleyin.
2. Dosyaya sag tiklayip paylasim ayarlarini acin.
3. Erisimi `linke sahip olan goruntuleyebilir` olarak ayarlayin.
4. Linki kopyalayip gorev panelindeki `proof link` alanina ekleyin.

Bu yontemde linkin gizli veya erisime kapali kalmadigindan emin olun.

## Aciklama Ornekleri

Genel gorevler icin:

```text
I completed the requested task. Since there is no screenshot upload field, I provided a public proof link where the evidence can be reviewed manually.
```

Takip gorevleri icin:

```text
I followed the requested QoreChain account/page and provided a public proof link for manual review.
```

Begeni veya etkilesim gorevleri icin:

```text
I completed the required engagement action and provided a public proof link showing the completed task.
```

GitHub gorevleri icin:

```text
I completed the requested GitHub action and provided a public GitHub link as proof for manual review.
```

## Dikkat Edilmesi Gerekenler

- Kanit linki inceleyici tarafindan acilabilmelidir.
- Ozel hesap, kapali grup veya giris isteyen sayfalar zayif kanit olabilir.
- Seed phrase, private key, cuzdan gizli bilgileri, e-posta veya telefon numarasi paylasmayin.
- Gerekirse ekran goruntusundeki hassas bilgileri kapatin.
- Kanit aciklamasinda abartili iddia veya onay garantisi kullanmayin.

## Kisa Ozet

Ekran goruntusu yukleme alani yoksa, kaniti linke donusturun. En onemli nokta, inceleyicinin linke tikladiginda gorevin yapildigini anlayabilmesidir.
