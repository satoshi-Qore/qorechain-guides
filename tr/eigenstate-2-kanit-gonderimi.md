# Eigenstate 2 Kanıt Gönderimi

Bu sayfa, Eigenstate 2 görevlerinde kanıt linki ve açıklama alanlarının nasıl daha düzenli kullanılabileceğini açıklar.

Görev formunda çoğu zaman yalnızca iki alan bulunur:

- proof link
- Describe what you did

Ekran görüntüsü yükleme alanı yoksa, kanıtın erişilebilir bir linke dönüştürülmesi gerekir.

## Temel Mantık

Kanıt, inceleyen kişinin görevin yapıldığını anlayabileceği bağlantı veya açıklamadır. Her görev doğrudan link üretmez. Bu yüzden kanıt formatı görevin türüne göre değişir.

## Kanıt Karar Ağacı

| Durum | Ne Kullanılır? |
|---|---|
| Public post yazıldı | Post linki |
| GitHub issue açıldı | Issue linki |
| GitHub PR gönderildi | PR linki |
| PR merge edildi | Merge edilmiş PR linki |
| Profil takip edildi | Profil veya resmi sayfa linki + kısa açıklama |
| Gönderi beğenildi | Gönderi linki + kısa açıklama |
| Watchlist / bookmark yapıldı | Kanıt sayfası veya ekran görüntüsü linki |
| Video izlendi | Video linki + kısa açıklama |
| Quiz tamamlandı | Resmi görev bağlamı + kısa açıklama |
| On-chain işlem yapıldı | Explorer veya işlem linki |

## Proof Link Alanı

Bu alana mümkün olduğunca doğrudan kanıt linki eklenmelidir.

Güçlü örnekler:

- GitHub issue linki
- GitHub PR linki
- Reddit post linki
- Medium yazı linki
- LinkedIn post linki
- Explorer transaction linki

Zayıf örnekler:

- Sadece ana sayfa linki
- Yanlış platform linki
- Erişilemeyen özel profil linki
- Görevle ilgisiz sosyal medya linki

## Ekran Görüntüsü Linke Nasıl Dönüştürülür?

Formda dosya yükleme alanı yoksa ekran görüntüsünü doğrudan yüklemek mümkün olmayabilir. Bu durumda güvenli ve erişilebilir bir yöntem kullanılmalıdır.

Uygun seçenekler:

- Kendi GitHub proof reposunda bir markdown sayfası hazırlamak
- GitHub issue veya gist kullanmak
- Kendi sosyal medya gönderinizde ekran görüntüsünü paylaşmak
- Erişimi açık bir belge veya sayfa linki kullanmak

Paylaşmadan önce hassas bilgileri mutlaka kapatın.

## Describe What You Did Alanı

Bu alan kısa, net ve abartısız olmalıdır. İnceleyen kişi görevin ne olduğunu ve sizin ne yaptığınızı hızlıca anlamalıdır.

Genel format:

```text
I completed the requested QoreChain task on [platform]. Since the task is reviewed manually, I submitted the related proof link and a short description for verification.
```

Takip görevi:

```text
I followed the official QoreChain page on [platform] and submitted the related page link for manual review.
```

Beğeni görevi:

```text
I liked the requested QoreChain post on [platform] and submitted the post link for manual review.
```

GitHub görevi:

```text
I completed the requested GitHub task and submitted the related repository, issue, or pull request link for manual review.
```

On-chain görev:

```text
I completed the requested on-chain action and submitted the related transaction or explorer link for manual review.
```

## Güçlü Kanıt Kontrol Listesi

- Link doğru platforma gidiyor.
- Link herkese açık veya inceleyen kişi tarafından erişilebilir.
- Açıklama görevi net anlatıyor.
- Hassas bilgi görünmüyor.
- Görev onaylanmadan "approved" denmiyor.
- Kanıt başka bir kullanıcının katkısını kendinizinmiş gibi göstermiyor.

## Hassas Bilgi Kontrolü

Şunları paylaşmayın:

- seed phrase
- private key
- cüzdan şifresi
- e-posta ve telefon
- özel mesajlar
- API key veya token
- sunucu IP veya erişim bilgileri

## Not

Bu rehber, topluluk tarafından hazırlanmış pratik bir kanıt gönderimi rehberidir. Görevlerin onaylanacağını garanti etmez. Resmi görev panelindeki güncel talimatlar esas alınmalıdır.
